Datasheetminer

Unified Rust binary: CLI + server-rendered web UI + REST API for industrial product specs


Rust Axum HTMX DynamoDB Gemini AI

Single Binary

CLI + web server + HTML frontend in one binary. No Node.js, no npm, no JavaScript framework.

Server-Rendered UI

Askama templates + HTMX for dynamic interactions. Zero build step for frontend.

AI Extraction

Gemini 2.5 Flash extracts specs from PDF and HTML datasheets automatically.

Unit Normalization

50+ unit conversions at compile time (mNm to Nm, kW to W, oz-in to Nm, etc).

Quickstart

1Prerequisites

Rust toolchain, AWS CLI configured with credentials, and a DynamoDB table.

2Create a dev DynamoDB table 
aws dynamodb create-table \
  --table-name products-dev \
  --attribute-definitions \
    AttributeName=PK,AttributeType=S \
    AttributeName=SK,AttributeType=S \
  --key-schema \
    AttributeName=PK,KeyType=HASH \
    AttributeName=SK,KeyType=RANGE \
  --billing-mode PAY_PER_REQUEST \
  --region us-east-1
3Build and run 
cd rust/
cargo build --release
./target/release/datasheetminer serve --port 8080 --table products-dev

Browse to http://localhost:8080 — products, datasheets, and management pages.

4Seed some test data (optional) 
# Scrape a datasheet
./target/release/datasheetminer scrape \
  --url "https://example.com/motor.pdf" \
  --type motor \
  --manufacturer "Delta" \
  --product-name "ECMA-C30804" \
  --table products-dev

# Or push from a JSON file
./target/release/datasheetminer push --file output.json --table products-dev
5Query from CLI 
./target/release/datasheetminer query --summary --table products-dev
./target/release/datasheetminer query --list --product-type motor --table products-dev
6Clean up 
aws dynamodb delete-table --table-name products-dev --region us-east-1

Web UI Pages

GET /                  Products table (sort, paginate, click for detail)
GET /datasheets        Datasheet list (create, delete)
GET /management        Delete by type, dedup, delete-all (with dry run)

All interactions use HTMX — partial page swaps, no full reloads.

JSON API

GET  /health
GET  /api/products?type=motor&limit=10
GET  /api/products/categories
GET  /api/products/summary
GET  /api/products/manufacturers
GET  /api/products/names
POST /api/products
GET  /api/datasheets
POST /api/datasheets

CLI Commands

datasheetminer serve     Start web server
datasheetminer scrape    Extract specs from a datasheet via Gemini AI
datasheetminer search    Discover datasheets via DuckDuckGo/Brave
datasheetminer push      Push JSON data to DynamoDB
datasheetminer query     Inspect database contents
datasheetminer delete    Delete products by type/criteria
datasheetminer dedup     Find and remove duplicates

Docker

# Build (single-stage Rust, no Node.js needed)
docker build -f rust/Dockerfile -t datasheetminer .

# Run
docker run -p 8080:8080 \
  -e AWS_ACCESS_KEY_ID=... \
  -e AWS_SECRET_ACCESS_KEY=... \
  -e AWS_DEFAULT_REGION=us-east-1 \
  datasheetminer serve --port 8080 --table products-dev
