Datasheet Miner

Extract, search, and compare industrial product specs from datasheets and product pages

Open App GitHub Self-Host

Data Sources

📄

PDF Datasheets

Upload or point to a PDF catalog. The datasheetminer CLI identifies spec pages, extracts structured data via Gemini AI, and pushes validated products to DynamoDB.

🌐

Product Webpages

The web-scraper CLI renders JS-heavy product pages in a headless browser, extracts JSON-LD structured data and HTML content, then runs the same AI extraction pipeline.

📝

Manual Entry

The web app's admin mode provides a product management UI for direct CRUD operations, bulk import, and datasheet upload via presigned S3 URLs.

Product Types

Motors

Brushless DC, AC servo, AC induction, and more. Voltage, current, power, torque, speed, encoder type, rotor inertia, IP rating.

Drives

Servo and VFD drives. Input/output voltage, power, switching frequency, I/O counts, fieldbus protocols, safety ratings.

Gearheads

Planetary, harmonic, cycloidal. Gear ratio, backlash, continuous/peak torque, input speed, torsional rigidity, service life.

Electric Cylinders

Stroke, push/pull force, continuous force, linear speed, positioning repeatability, lead screw pitch.

Robot Arms

Payload, reach, pose repeatability, max TCP speed, number of axes, per-axis torque and speed.

Extensible

Add a Pydantic model in datasheetminer/models/ and it auto-discovers in all CLIs and the web app. No config changes needed.

Features

🔍

Advanced Filtering

Filter by any spec attribute with IS/NOT modes, range sliders, and multi-select. Sort by multiple columns simultaneously.

⚙

Gear Ratio Computation

Auto-compute per-motor gear ratios from torque and speed filters, showing what gearing each motor needs to meet your requirements.

💻

CLI Tools

Five CLI commands: datasheetminer for PDFs, web-scraper for webpages, page-finder for spec-page detection, dsm-agent for batch workflows, dsm for queries.

📱

Mobile Responsive

Collapsible filters on mobile with touch-friendly targets. Dark/light theme toggle. Desktop-first with a snug mobile layout.

Quick Start

# Clone and install
git clone https://github.com/jimothyjohn/datasheetminer.git
cd datasheetminer
uv sync

# Extract specs from a PDF datasheet
uv run datasheetminer \
  --url "https://example.com/motor-catalog.pdf" \
  --type motor --manufacturer "Acme" --product-name "X100"

# Scrape a product webpage (JS-rendered)
uv run web-scraper \
  --url "https://shop.example.com/products/X100" \
  --type motor --manufacturer "Acme" --product-name "X100"

# Query the database
uv run dsm find --type motor \
  --where "rated_power>=1000" --sort "rated_torque:desc"

# Start the web app
cd app && npm install && npm run dev

API

Endpoint Description
GET /health Health check with status, mode, and timestamp
GET /api/v1/search Full-text search with filters, sorting, and pagination
GET /api/products List products (filter with ?type=motor)
GET /api/products/summary Counts by product type
GET /api/products/categories Available product types with counts
GET /api/products/manufacturers Unique manufacturer list
GET /api/datasheets Datasheet source entries
POST /api/upload Queue a datasheet for AI processing (admin)

Architecture

Extraction (Python)

Gemini AI extracts specs as CSV with unit-in-header columns. Parsed locally into value;unit compact strings, validated by Pydantic models and unit-family rules, quality-scored before storage.

Frontend

React + TypeScript + Vite. Deployed to S3 behind CloudFront. Two modes: admin (full CRUD) and public (read-only search and filtering).

Backend

Express.js on AWS Lambda via API Gateway. REST API with search, product CRUD, datasheet management, and upload pipeline.

Data

DynamoDB single-table design with PK=PRODUCT#TYPE, SK=PRODUCT#UUID. S3 for PDF storage. Deterministic UUIDs for deduplication across sources.
